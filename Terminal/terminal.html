Essential Terminal Commands

September 17, 2025
I find using the terminal fun, and one of the joys of using Linux. It often makes things easier. This post is a few notes on useful commands that I'm sharing in case you find them useful, like they were for me.

Navigation and File System Commands

1. pwd - Print Working Directory

The most fundamental command - shows you exactly where you are in the file system. 

$ pwd
/home/xala/projects/my-blog

Use this whenever you're lost or need to confirm your current location before running other commands.

Lists files and directories in the current location. This command has numerous useful flags:

A flag in terminal commands is an option or modifier (usually preceded by - or --) that changes the behavior of a command or adds specific functionalities.

# Combine flags for detailed view including hidden files / combinar flags para vista detallada incluyendo archivos ocultos

# Human-readable file sizes / tama침os de archivos legibles por humanos

# Sort by modification time (newest first) / ordenar por tiempo de modificaci칩n (el m치s nuevo primero)

Note: human-readable files provide clarity. They allow you to see at a glance what a file contains without needing specialized software. Example: a .txt, .csv, or .json file can be opened and read easily, in contrast to a .bin or .exe.

3. cd - Change Directory

Navigate between directories efficiently:

# or simply / o simplemente

File Operations Commands

4. mkdir - Make Directory

Create new directories:

# Create a single directory / crear un solo directorio
$ mkdir my-project

# Create multiple directories / crear m칰ltiples directorios

# Create directory with specific permissions / crear directorio con permisos espec칤ficos
$ mkdir -m 755 public-folder

The command mkdir is used to create directories. The -m flag (mode) sets the permissions of the new directory at the time of creation, using the same numeric (octal) format as chmod. Example: mkdir -m 700 private creates a directory named 'private' with permissions rwx------, while mkdir -m 755 public creates a directory named 'public' with permissions rwxr-xr-x.

In Unix-like systems, file and directory permissions are represented with the letters r, w, and x. 'r' means read (the ability to view the contents of a file or list the contents of a directory). 'w' means write (the ability to modify a file or create/delete items in a directory). 'x' means execute (the ability to run a file as a program, or to enter a directory and access its contents).

File permissions are shown in a 10-character string, for example: rwxr-xr--. The first character indicates the type (e.g., - for a file, d for a directory). The next nine characters are grouped in sets of three: the first set (rwx) applies to the owner, the second set (r-x) applies to the group, and the third set (r--) applies to others. Each position represents a permission: r for read, w for write, and x for execute.

File permissions are divided into three levels: owner, group, and others. The owner is the user who created the file and has the main control over it. The group consists of users who belong to the same group as the file; they have the permissions assigned to the group (read, write, execute). Others are all users who are neither the owner nor members of the group; their permissions are usually more restricted for security.

Example of file permissions: rwxr-xr--. The first three characters 'rwx' apply to the owner (read, write, execute). The next three 'r-x' apply to the group (read and execute, no write). The last three 'r--' apply to others (read only).

For temporal directories

# Create a temporary directory / crear un directorio temporal

# Create a temporary directory with a specific pattern / crear un directorio temporal con un patr칩n espec칤fico
$ mktemp -d /tmp/mytempdir.XXXX

# Remove a temporary directory manually / eliminar manualmente un directorio temporal
$ rm -r /tmp/mytempdir.XYZ123

Note about duration

  • Directories in /tmp usually last until system reboot
  • Directories in /var/tmp may last longer depending on system configuration

Recursively

The -r (recursive) flag in Unix/Linux commands allows the command to operate on directories and their contents, including all subdirectories and files. It is commonly used with commands like cp, rm, and chmod to apply actions recursively. 

# Copy a directory and all its contents / copiar un directorio y todo su contenido
$ cp -r carpeta_original carpeta_copia

# Remove a directory and all its contents / eliminar un directorio y todo su contenido
$ rm -r carpeta_para_borrar

# Change permissions of a directory and all files inside / cambiar permisos de un directorio y todos los archivos dentro
$ chmod -R 755 carpeta

5. touch - Create Files

Create empty files or update file timestamps:

# Create multiple files / crear m칰ltiples archivos
$ touch style.css script.js README.md

# Update timestamp of existing file / actualizar marca de tiempo de archivo existente

Updating a file's timestamp means modifying the information that indicates when it was last created, modified, or accessed. Each file has three main timestamps: atime (last access), mtime (last modification), and ctime (last metadata change). 



# Check the timestamps of a file / verificar las marcas de tiempo de un archivo
$ stat archivo.txt

6. cp - Copy Files and Directories

# Copy a file
$ cp source.txt destination.txt

# Copy a file to another directory
$ cp file.txt /path/to/directory/

# Copy directory recursively
$ cp -r source-directory/ destination-directory/

# Copy with verbose output
$ cp -v file.txt backup/

# Copy and preserve file attributes
$ cp -p original.txt backup.txt

7. mv - Move/Rename Files

# Rename a file
$ mv old-name.txt new-name.txt

# Move file to another directory
$ mv file.txt /path/to/directory/

# Move and rename simultaneously
$ mv old-file.txt /new/location/new-name.txt

# Move multiple files to a directory
$ mv file1.txt file2.txt file3.txt /destination/

Keyboard Shortcuts

  • Ctrl+C: Stop/cancel current command
  • Ctrl+D: Exit terminal/logout
  • Ctrl+L: Clear screen (same as 'clear' command)
  • Ctrl+A: Move cursor to beginning of line
  • Ctrl+E: Move cursor to end of line
  • Tab: Auto-complete commands and filenames

Best Practices and Safety Tips

Safety Guidelines

  • Always double-check commands before pressing Enter, especially with rm, mv, and chmod
  • Use the -i flag with destructive commands for confirmation prompts
  • Test commands in a safe directory first
  • Keep backups of important files
  • Use version control (Git) for code projects
