Essential Terminal Commands

September 17, 2025
I find using the terminal fun, and one of the joys of using Linux. It often makes things easier. This post is a few notes on useful commands that I'm sharing in case you find them useful, like they were for me.

Navigation and File System Commands

1. pwd - Print Working Directory

The most fundamental command - shows you exactly where you are in the file system. 

$ pwd
/home/xala/projects/my-blog

Use this whenever you're lost or need to confirm your current location before running other commands.

2. ls - List Directory Contents

Lists files and directories in the current location. This command has numerous useful flags:

A flag in terminal commands is an option or modifier (usually preceded by - or --) that changes the behavior of a command or adds specific functionalities. 

# Basic listing / listado básico
$ ls

# Detailed listing with permissions, sizes, dates /listado detallado con permisos, tamaños, fechas
$ ls -l

# Show hidden files (starting with .) / mostrar archivos ocultos (que comienzan con .)
$ ls -a

# Combine flags for detailed view including hidden files / combinar flags para vista detallada incluyendo archivos ocultos
$ ls -la

# Human-readable file sizes / tamaños de archivos legibles por humanos
$ ls -lh

# Sort by modification time (newest first) / ordenar por tiempo de modificación (el más nuevo primero)
$ ls -lt

Note: human-readable files provide clarity. They allow you to see at a glance what a file contains without needing specialized software. Example: a .txt, .csv, or .json file can be opened and read easily, in contrast to a .bin or .exe.

3. cd - Change Directory

Navigate between directories efficiently:

# Go to a specific directory / ir a directorio específico
$ cd /path/to/directory

# Go to home directory / ir al directorio home
$ cd ~
# or simply / o simplemente
$ cd

# Go up one level / subir un nivel
$ cd ..

# Go up two levels / subir dos niveles
$ cd ../..

# Go to the previous directory / ir al directorio anterior
$ cd -

# Navigate to subdirectory
$ cd Documents/projects

Pro Tip: Tab Completion

Use the Tab key to auto-complete directory and file names. This saves time and prevents typos. Press Tab twice to see all available options if there are multiple matches.

It's amazing what you can discover ^^

File Operations Commands

4. mkdir - Make Directory

Create new directories:

# Create a single directory / crear un solo directorio
$ mkdir my-project

# Create multiple directories / crear múltiples directorios
$ mkdir dir1 dir2 dir3

# Create parent directories as needed / crear directorios padre según sea necesario
$ mkdir -p projects/web-app/src/components

# Create directory with specific permissions / crear directorio con permisos específicos
$ mkdir -m 755 public-folder

The command mkdir is used to create directories. The -m flag (mode) sets the permissions of the new directory at the time of creation, using the same numeric (octal) format as chmod. Example: mkdir -m 700 private creates a directory named 'private' with permissions rwx------, while mkdir -m 755 public creates a directory named 'public' with permissions rwxr-xr-x.

In Unix-like systems, file and directory permissions are represented with the letters r, w, and x. 'r' means read (the ability to view the contents of a file or list the contents of a directory). 'w' means write (the ability to modify a file or create/delete items in a directory). 'x' means execute (the ability to run a file as a program, or to enter a directory and access its contents).

File permissions are shown in a 10-character string, for example: rwxr-xr--. The first character indicates the type (e.g., - for a file, d for a directory). The next nine characters are grouped in sets of three: the first set (rwx) applies to the owner, the second set (r-x) applies to the group, and the third set (r--) applies to others. Each position represents a permission: r for read, w for write, and x for execute.

File permissions are divided into three levels: owner, group, and others. The owner is the user who created the file and has the main control over it. The group consists of users who belong to the same group as the file; they have the permissions assigned to the group (read, write, execute). Others are all users who are neither the owner nor members of the group; their permissions are usually more restricted for security.

Example of file permissions: rwxr-xr--. The first three characters 'rwx' apply to the owner (read, write, execute). The next three 'r-x' apply to the group (read and execute, no write). The last three 'r--' apply to others (read only).

For temporal directories

# Create a temporary directory / crear un directorio temporal
$ mktemp -d

# Create a temporary directory with a specific pattern / crear un directorio temporal con un patrón específico
$ mktemp -d /tmp/mytempdir.XXXX

# Remove a temporary directory manually / eliminar manualmente un directorio temporal
$ rm -r /tmp/mytempdir.XYZ123

Note about duration

  • Directories in /tmp usually last until system reboot
  • Directories in /var/tmp may last longer depending on system configuration

Recursively

The -r (recursive) flag in Unix/Linux commands allows the command to operate on directories and their contents, including all subdirectories and files. It is commonly used with commands like cp, rm, and chmod to apply actions recursively. 

# Copy a directory and all its contents / copiar un directorio y todo su contenido
$ cp -r carpeta_original carpeta_copia

# Remove a directory and all its contents / eliminar un directorio y todo su contenido
$ rm -r carpeta_para_borrar

# Change permissions of a directory and all files inside / cambiar permisos de un directorio y todos los archivos dentro
$ chmod -R 755 carpeta

5. touch - Create Files

Create empty files or update file timestamps:

# Create a new empty file / crear un nuevo archivo vacío
$ touch index.html

# Create multiple files / crear múltiples archivos
$ touch style.css script.js README.md

# Update timestamp of existing file / actualizar marca de tiempo de archivo existente
$ touch existing-file.txt

Updating a file's timestamp means modifying the information that indicates when it was last created, modified, or accessed. Each file has three main timestamps: atime (last access), mtime (last modification), and ctime (last metadata change). 



# Check the timestamps of a file / verificar las marcas de tiempo de un archivo
$ stat archivo.txt

6. cp - Copy Files and Directories

# Copy a file
$ cp source.txt destination.txt

# Copy a file to another directory
$ cp file.txt /path/to/directory/

# Copy directory recursively
$ cp -r source-directory/ destination-directory/

# Copy with verbose output
$ cp -v file.txt backup/

# Copy and preserve file attributes
$ cp -p original.txt backup.txt

7. mv - Move/Rename Files

# Rename a file
$ mv old-name.txt new-name.txt

# Move file to another directory
$ mv file.txt /path/to/directory/

# Move and rename simultaneously
$ mv old-file.txt /new/location/new-name.txt

# Move multiple files to a directory
$ mv file1.txt file2.txt file3.txt /destination/

8. rm - Remove Files and Directories

⚠️ Be very careful with this command - deleted files are usually not recoverable! 

# Remove a file / eliminar un archivo
$ rm file.txt

# Remove multiple files / eliminar múltiples archivos
$ rm file1.txt file2.txt file3.txt

# Remove directory and its contents recursively /eliminar directorio y su contenido recursivamente
$ rm -r directory/

# Force removal (no prompts) /eliminación forzada (sin avisos)
$ rm -f file.txt

# Interactive removal (asks for confirmation) /eliminación interactiva (pide confirmación)
$ rm -i important-file.txt

# Remove empty directory / eliminar directorio vacío
$ rmdir empty-directory/
Mastering terminal commands is essential for efficient development workflows

File Content Commands

9. cat - Display File Contents

# Display entire file content / mostrar todo el contenido del archivo
$ cat file.txt

# Display multiple files / mostrar múltiples archivos
$ cat file1.txt file2.txt

# Display with line numbers /   mostrar con números de línea
$ cat -n file.txt

# Create a new file with content / crear un nuevo archivo con contenido
$ cat > new-file.txt
Type your content here
Press Ctrl+D to save and exit

10. less/more - View File Contents Page by Page

Better for viewing large files:

# View file with less (recommended) / ver archivo con less (recomendado)
$ less large-file.txt

# Navigation in less:
# - Space or f: forward one page
# - b: backward one page
# - /pattern: search for pattern
# - q: quit

# View with more (simpler version) /ver con more (versión más simple)
$ more file.txt

11. head/tail - View Beginning/End of Files

# Show first 10 lines (default) / mostrar primeras 10 líneas (por defecto)
$ head file.txt

# Show first 20 lines / mostrar primeras 20 líneas
$ head -n 20 file.txt

# Show last 10 lines / mostrar últimas 10 líneas
$ tail file.txt

# Show last 15 lines / mostrar últimas 15 líneas
$ tail -n 15 file.txt

# Follow file changes (great for logs) / seguir cambios en el archivo (genial para logs)
$ tail -f logfile.txt

Search and Filter Commands

12. grep - Search Text Patterns

One of the most powerful text processing commands:

# Search for a pattern in a file / buscar un patrón en un archivo
$ grep "error" logfile.txt

# Case-insensitive search /búsqueda sin distinción de mayúsculas
$ grep -i "ERROR" logfile.txt

# Search recursively in directories / buscar recursivamente en directorios
$ grep -r "function" src/

# Show line numbers / mostrar números de línea
$ grep -n "TODO" *.js

# Search for whole words only / buscar solo palabras completas
$ grep -w "test" file.txt

# Invert match (show lines that don't match) / coincidencia invertida (mostrar líneas que no coinciden)
$ grep -v "debug" logfile.txt

# Count matching lines / contar líneas coincidentes
$ grep -c "error" logfile.txt

13. find - Search for Files and Directories

# Find files by name / Encontrar archivo por nombre
$ find . -name "*.js"

# Find directories only / Encontrar solo directorios
$ find . -type d -name "node_modules"

# Find files by size / Encontrar archivos por tamaño
$ find . -size +100M

# Find files modified in last 7 days / Encontrar archivos modificados en los últimos 7 días
$ find . -mtime -7

# Find and execute command on results / Encontrar y ejecutar comando en los resultados
$ find . -name "*.log" -delete

# Find files with specific permissions / Encontrar archivos con permisos específicos
$ find . -perm 644

System Information Commands

14. ps - Process Status

# Show running processes for current user / mostrar procesos en ejecución para el usuario actual
$ ps

# Show all processes with detailed info / mostrar todos los procesos con información detallada
$ ps aux

# Show processes in tree format / mostrar procesos en formato de árbol
$ ps -ef --forest

# Find specific process / encontrar proceso específico
$ ps aux | grep "nginx"

15. top/htop - System Activity Monitor

# Show real-time system statistics / mostrar estadísticas del sistema en tiempo real
$ top

# Enhanced version (if installed) / versión mejorada (si está instalada)
$ htop

# Sort by memory usage in top / ordenar por uso de memoria en top
# Press Shift+M while top is running

# Sort by CPU usage in top / ordenar por uso de CPU en top
# Press Shift+P while top is running

16. df/du - Disk Space Usage

# Show filesystem disk space usage / mostrar uso del espacio en disco del sistema de archivos
$ df -h

# Show directory size / mostrar tamaño del directorio
$ du -h directory/

# Show size of current directory / mostrar tamaño del directorio actual
$ du -sh .

# Show sizes of all files and directories / mostrar tamaños de todos los archivos y directorios
$ du -ah

Network and Connectivity Commands

17. wget/curl - Download Files

# Download file with wget / descargar archivo con wget
$ wget https://example.com/file.zip

# Download with curl / descargar con curl
$ curl -O https://example.com/file.zip

# Download and save with different name / descargar y guardar con nombre diferente
$ curl https://example.com/data.json -o my-data.json

# Make HTTP request and see response / hacer solicitud HTTP y ver respuesta
$ curl -v https://api.example.com/users

18. ssh - Secure Shell

# Connect to remote server / conectar a servidor remoto
$ ssh user@hostname

# Connect on specific port / conectar en puerto específico
$ ssh -p 2222 user@hostname

# Copy files to remote server / copiar archivos a servidor remoto
$ scp file.txt user@hostname:/path/to/destination/

# Copy files from remote server / copiar archivos desde servidor remoto
$ scp user@hostname:/path/to/file.txt ./local-directory/

Advanced Tips and Combinations

Pipe Operations

Combine commands using pipes (|) to create powerful workflows: 

# Count lines in a file / contar líneas en un archivo
$ cat file.txt | wc -l

# Find and count specific pattern / encontrar y contar patrón específico
$ grep "error" logfile.txt | wc -l

# Show only unique lines / mostrar solo líneas únicas
$ sort file.txt | uniq

# Find largest files in directory / encontrar archivos más grandes en el directorio
$ ls -la | sort -k5 -nr | head -10
# or / o
$ls -lS | head -n 10  


# Monitor real-time log for specific pattern / monitorear log en tiempo real para patrón específico
$ tail -f access.log | grep "404"

Command History and Shortcuts

# Show command history / mostrar historial de comandos
$ history

# Re-run last command / volver a ejecutar el último comando
$ !!

# Re-run specific command from history / volver a ejecutar comando específico del historial
$ !123

# Search command history / buscar en el historial de comandos
$ history | grep "git"

# Clear command history / limpiar historial de comandos
$ history -c

Keyboard Shortcuts

  • Ctrl+C: Stop/cancel current command
  • Ctrl+D: Exit terminal/logout
  • Ctrl+L: Clear screen (same as 'clear' command)
  • Ctrl+A: Move cursor to beginning of line
  • Ctrl+E: Move cursor to end of line
  • Tab: Auto-complete commands and filenames

Best Practices and Safety Tips

Safety Guidelines

  • Always double-check commands before pressing Enter, especially with rm, mv, and chmod
  • Use the -i flag with destructive commands for confirmation prompts
  • Test commands in a safe directory first
  • Keep backups of important files
  • Use version control (Git) for code projects
